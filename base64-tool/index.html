🤔 Base64 인코딩 관련 Q&A
Base64 인코딩이란 무엇인가요?
컴퓨터 분야에서 쓰이는 Base64(베이스 64) 코딩 기법은 8비트 이진 데이터(예: 이미지 파일, 실행파일, 일반 문자열 등)를 화면에 표시되는 문자 코드표(ASCII)의 일반적인 64개 아스키 문자로 변환하여 나타내는 방식입니다. 이렇게 변환하면 시스템 독립적으로 데이터를 안전하게 전송하거나 저장할 수 있습니다.
Base64는 암호화 방식인가요?
아닙니다. Base64는 '인코딩(Encoding)' 방식일 뿐, 보안을 위한 '암호화(Encryption)'가 아닙니다. 누구나 Base64 문자열을 원본 데이터로 다시 디코딩할 수 있으므로, 비밀번호나 개인정보 같은 민감한 데이터는 Base64로만 처리해서 노출하면 안전하지 않습니다. 반드시 강력한 암호화 알고리즘 처리를 별도로 해야 합니다.
이미지를 Base64로 언제 주로 변환하나요?
HTML 문서 내에 작은 크기의 로고 이미지나 아이콘을 삽입할 때 주로 사용됩니다.
문서 내에 <img src="data:image/png;base64,...(데이터)..." /> 형태로 포함시키면 별도의 이미지 파일을 다운로드하기 위한 추가적인 네트워크 HTTP 요청이 발생하지 않아 페이지 로딩 속도를 일부 개선하는 데 도움을 줍니다. 단, 이미지 용량이 너무 크면 문서의 텍스트 크기가 과도하게 커질 수 있으므로 수 킬로바이트 이하의 소형 이미지에 권장됩니다.
